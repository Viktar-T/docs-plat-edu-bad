## Architecture Overview

The platform is delivered through two coordinated layers:

- **Official Landing (`oze.zut.edu.pl`)** â€” Astro static site hosted on the universityâ€™s Apache/PHP infrastructure. It provides faculty and project information and routes visitors to the interactive platform.
- **Research & Education Platform (VPS)** â€” Docker Compose orchestrates InfluxDB, Node-RED, Grafana, an Express-based API Gateway, and a React frontend. This environment hosts synthetic scenarios today and is engineered for future physical integrations.

## Main Components

| Component | Description | Technology |
| --- | --- | --- |
| Astro Landing Page (`oze.zut.edu.pl`) | Official public site with institutional branding, documentation, and routing to the platform. | Astro, static hosting (Apache/PHP) |
| React Frontend (Platform UI) | Primary user interface on the VPS, consuming the API Gateway and embedding authorized Grafana resources. | React, Vite build |
| API Gateway | Unified external API, request routing, access control, and service abstraction for frontend clients. | Node.js, Express |
| Node-RED | Orchestrates simulated and live data flows, educational scenarios, and integrations with external sources. | Node-RED |
| InfluxDB | Central time-series datastore for telemetry generated by Node-RED scenarios. | InfluxDB 2.x |
| Grafana | Dashboarding layer providing curated analytics and research views over InfluxDB datasets. | Grafana |
| Docker & Docker Compose | Manages container lifecycle, networking, persistence, and deployment across the VPS environment. | Docker Engine, docker-compose |

## Data & Request Flow

1. Simulated or curated datasets enter **Node-RED**, where flows validate, transform, and route the data.
2. Node-RED persists processed telemetry into **InfluxDB** using measurement-specific buckets.
3. **Grafana** queries InfluxDB to expose dashboards, panels, and reports for research and teaching.
4. The **API Gateway** aggregates read/write operations, exposes curated endpoints, and mediates access to Node-RED hooks and InfluxDB queries.
5. The **React frontend** relies exclusively on the API Gateway and configured Grafana embeds to deliver the user experience.
6. Users land on `oze.zut.edu.pl`, review information, and launch the VPS-hosted platform for interactive exploration.

## Key Features

- **Layered delivery** separating public communication (`oze.zut.edu.pl`) from the secure VPS platform.
- **Centralized API Gateway** providing a consistent surface area, security policies, and integration logic.
- **Modular containers** for Node-RED, InfluxDB, Grafana, API Gateway, and the React frontend, orchestrated via Docker Compose.
- **Education-first data** with synthetic scenarios ready to transition to laboratory-grade telemetry.
- **Extensible blueprint** enabling future services such as MQTT brokers, authentication providers, or additional microservices without rework.
- **Operational readiness** through environment parity, reproducible deployments, and clear responsibility boundaries between hosting layers.
